## Important: Users Can Subscribe to Individual Moons

A user might have Forge but NOT Quill. They might have the full bundle (all 13) or just one.

**Your app must check for the specific Moon it uses ‚Äî not just "are they subscribed."**

For example:
- Forge Builder checks for `forge` ‚Äî if the user doesn't have Forge, they can't use it
- EzQuill checks for `quill` ‚Äî Forge access doesn't help them here
- A bundle subscriber gets all 13, so they pass every check

Each Moon has its own message quota. Using Forge does NOT eat into Quill's messages. They're independent.

---

## Security Layers (Built In)

1. **Domain Lock** ‚Äî Each key only works from its registered domain. Wrong domain = rejected.
2. **IP Allowlist** ‚Äî If IPs are set, only those server IPs can use the key. Wrong IP = rejected.
3. **Rate Limiting** ‚Äî Default 60 requests/minute. Exceeding 3x the limit auto-locks the key.
4. **Auto-Lock** ‚Äî Suspicious activity locks the key. Only admin can unlock via /moon-api-admin.

If a key is stolen, the thief cannot use it from a different domain or IP. It's useless to them.

## Endpoints

### 1. Check If User Has Moon Access

```
GET /api/moon/verify?userId=USER_ID&moon=MOON_NAME
Header: x-moon-api-key: YOUR_KEY
```

**Response:**
```json
{
  "active": true,
  "isBundle": false,
  "moonName": "forge",
  "messagesRemaining": 87,
  "expiresAt": "2026-05-14T00:00:00.000Z",
  "isFreeUser": false
}
```

### 2. Get All Subscriptions for a User

```
GET /api/moon/subscriptions?userId=USER_ID
Header: x-moon-api-key: YOUR_KEY
```

**Response:**
```json
{
  "userId": "abc123",
  "isBundle": true,
  "activeMoons": ["road_dog", "hawk", "forge", "ledger", ...],
  "messagesRemaining": { "road_dog": 450, "hawk": 480, ... },
  "subscriptions": [
    {
      "moonName": "bundle",
      "isBundle": true,
      "expiresAt": "2026-05-14T00:00:00.000Z",
      "createdAt": "2026-04-14T00:00:00.000Z"
    }
  ]
}
```

### 3. Deduct a Message (After AI Responds)

```
POST /api/moon/deduct
Header: x-moon-api-key: YOUR_KEY
Content-Type: application/json

Body:
{
  "userId": "abc123",
  "moonName": "forge",
  "count": 1
}
```

**Response:**
```json
{
  "success": true,
  "messagesRemaining": 86,
  "moonName": "forge",
  "appSource": "Forge Builder"
}
```

### 4. Full User Status (All Moons at Once)

```
GET /api/moon/user/USER_ID/status
Header: x-moon-api-key: YOUR_KEY
```

**Response:**
```json
{
  "userId": "abc123",
  "hasSubscription": true,
  "isBundle": true,
  "activeMoons": ["road_dog", "hawk", "forge", ...],
  "usage": {
    "forge": { "used": 14, "limit": 500, "remaining": 486 },
    "hawk": { "used": 3, "limit": 500, "remaining": 497 }
  },
  "isFreeUser": false,
  "freeMessagesTotal": 5
}
```

### 5. Add Bonus Messages

```
POST /api/moon/add-messages
Header: x-moon-api-key: YOUR_KEY
Content-Type: application/json

Body:
{
  "userId": "abc123",
  "moonName": "forge",
  "count": 50
}
```

## How It Works for Your App

**Step 1: User opens your app** (they already have a Town Square account)

**Step 2: Check for the SPECIFIC Moon your app uses**
```
GET /api/moon/verify?userId=USER_ID&moon=forge
```
This returns `"active": true` or `"active": false`.

- If the user has that Moon individually OR has the full bundle ‚Üí active = true
- If the user has other Moons but NOT this one ‚Üí active = false
- If the user is on free trial ‚Üí they get 5 free messages, then it's done

**Step 3: If active, let them use the AI**
After the AI responds, deduct from THAT Moon's quota:
```
POST /api/moon/deduct
Body: { "userId": "...", "moonName": "forge", "count": 1 }
```

**Step 4: If not active, send them to subscribe**
Link: `https://thepeoplestownsq.com/ai-education`
They can buy just the Moon they need ($1-$4/month) or the full bundle ($25/month).

### Example: Forge Builder App Flow
```
1. User opens Forge Builder
2. App calls: GET /api/moon/verify?userId=abc123&moon=forge
3. Response: { "active": true, "messagesRemaining": 87 }
4. User writes code with Forge AI ‚Üí Forge responds
5. App calls: POST /api/moon/deduct { userId: "abc123", moonName: "forge", count: 1 }
6. Next time: messagesRemaining = 86
```

### Example: User Has Quill But NOT Forge
```
1. User opens Forge Builder
2. App calls: GET /api/moon/verify?userId=abc123&moon=forge
3. Response: { "active": false, "messagesRemaining": 0 }
4. App shows: "You need a Forge subscription. Subscribe at thepeoplestownsq.com/ai-education"
   (Their Quill subscription is irrelevant here ‚Äî each Moon is independent)
```

### Example: Bundle Subscriber
```
1. User opens ANY app (Forge, Quill, Call Guardian, etc.)
2. App calls: GET /api/moon/verify?userId=abc123&moon=forge (or any moon)
3. Response: { "active": true, "isBundle": true, "messagesRemaining": 486 }
4. Bundle subscribers pass every Moon check. 500 messages/month per Moon.
```

## Error Responses

| Code | Meaning |
|------|---------|
| 401 | Missing or invalid API key |
| 403 | Domain not authorized, IP not authorized, key locked, or no messages remaining |
| 429 | Rate limit exceeded (wait and retry) |
| 400 | Bad request (missing fields, invalid moon name) |
